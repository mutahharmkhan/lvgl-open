### lv_test_wait

Emulate a delay. It's not real delay, but it tricks LVGL to think that the required time has been elapsed. <ApiLink name="lv_timer_handler" /> is called after each millisecond, meaning all the events will be fired inside this function. At the end the animations and display will be also updated.

```c
void lv_test_wait(uint32_t ms)
```

<span className="sr-only">Parameters</span>

| Name | Type | Description |
|------|------|-------------|
| `ms` | <ApiLink name="uint32_t" /> | the number of milliseconds to pass |


</ApiMember>
